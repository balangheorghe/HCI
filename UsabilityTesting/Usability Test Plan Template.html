Test Plan for: ARC

Persona Type Tested: Primary

Context: ARC HQ
Duration: 15 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • User is able to login using the android application
  • User is able to set his preferences
  • User is able to select a cloth
  • User is able to customize the selected cloth
  • User is able to visualize himselft wearing the cloth
  • User is able to order the cloth
Scenario

The user visits an augmented reality enabled phisical store. He directly goes into a fitting room. Based on his phone and its internal sensors, the user gets authenticated, and, taking into account the current season, weather, user body type, body sizes and his personal preferences for clothes, various outfits and pieces of clothing are presented to him on a mirror like, human sized touch screen. He then chooses what he likes and the display renders him already wearing what he choosed.

Based on the render, he thinks the item he is trying out will be way to tight, so by adjust a few sliders he makes the item a little bit less firm. The item is rendered in real time, while he is turning to his side to see the item from multiple angles.

The user likes how the item fits him, so he decides to buy one for his size. He presses the buy button, and a few days later the item is delivered to his doorstep.

Tasks
  • User downloads and installs the ARC Android Application - 1 minute
  • User performs a login using his google account - 1 minute
  • User scans the Room NFC and is being rendered - 1 minute
  • User sets his preferences regarding the clothes - 2 minutes
  • User navigates between available clothes and select the favourite one - 2 minutes
  • User visualizes a render of the cloth on himself - 1 minutes
  • User tries another clothes - 2 minute
  • User decides what cloth to buy - 2 minutes
  • User proceeds to the cart section, places the order and finalize the command - 3 minutes
Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • How was the experience?
  • How would you describe the application to your friends?
  • Would you use it?
  • How often would you relly on it?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".
References
  • Fluid User Testing;
  • usability.gov Templates;
  • Handbook of Usability Testing by Jeffrey Rubin;
  • A Practical Guide to Usability Testing by Joseph S. Dumas and Janice C. Redish.
